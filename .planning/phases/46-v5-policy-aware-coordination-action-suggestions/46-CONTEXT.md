# Context 46: `v5` Policy-Aware Coordination Action Suggestions

## Why This Exists

Phase 45 exposed bounded coordination actions to richer operator surfaces, but
those surfaces still need help choosing the right one under pressure. The next
slice should add compact, policy-aware suggestions without turning coordination
into silent automation.

## Inputs

- phase 40 dashboard views
- phase 41 drilldown views
- phase 42 watch and alert views
- phase 43 reusable coordination change feeds
- phase 44 UI-friendly coordination feed surfaces
- phase 45 coordination action surfaces

## Desired Outcome

Expose bounded coordination action suggestions derived from the current inbox,
recovery, policy, and history state while preserving explicit operator choice.
