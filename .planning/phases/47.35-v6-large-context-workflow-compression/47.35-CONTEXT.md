# Phase 47.35: v6 Large-Context Workflow Compression — Context

## Why This Exists

Some tasks legitimately need huge context, but not on every intermediate turn.
Memd should split those jobs into reusable layers so short and medium sessions
stay competitive while still supporting deliberate broad passes when they add
value.

## Decisions

- Use a layered workflow: global brief, glossary, entity/reference sheets,
  chunk-local windows, and reconciliation passes.
- Preserve provenance and cross-chunk traceability.
- Keep the first slice focused on global coherence instead of giant-context
  replay.
- Treat broad-context passes as deliberate, not default.

## Discretion Areas

- How much of the long-form workflow should be automatic versus operator-driven.
- How much of the layered context should be rendered in the CLI versus stored
  in files.
- Whether chunk-local windows should be derived from document structure or
  from operator hints first.

## Deferred Ideas

- Dedicated translation/synthesis UI.
- More advanced document segmentation.
- Adaptive chunk sizing based on observed performance.
