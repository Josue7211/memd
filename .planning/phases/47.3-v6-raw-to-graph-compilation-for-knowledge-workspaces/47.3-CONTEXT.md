# Phase 47.3: v6 Raw-to-Graph Compilation for Knowledge Workspaces — Context

## Why This Exists

Raw folders should compile into reusable knowledge artifacts so repeated
cold-reads stop burning tokens. The filesystem-first path should work before
any semantic backend is required.

## Decisions

- Prefer compiled graph/wiki outputs before rereading raw files when the
  compiled lane is fresh.
- Preserve provenance classes on graph edges: extracted, inferred, ambiguous.
- Keep the first slice filesystem-first so Obsidian-only setups benefit.
- Treat compiled artifacts as durable knowledge workspace outputs, not just
  transient cache files.

## Discretion Areas

- How much graph detail should appear in the default compiled output.
- Whether compiled outputs should be refreshed on every import or only when the
  source registry says a source changed.
- How much of the graph provenance should be visible in the main CLI versus
  deeper drilldowns.

## Deferred Ideas

- Semantic backend integration.
- Automatic entity-resolution tuning.
- Any richer graph UI beyond the filesystem artifacts.
