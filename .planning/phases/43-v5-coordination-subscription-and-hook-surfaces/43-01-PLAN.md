# Plan 43-01: `v5` Coordination Subscription and Hook Surfaces

## Goal

Let other surfaces consume compact coordination change events without
reimplementing watch-loop diff logic.

## Steps

1. Extract the current coordination change detection into a reusable response shape.
2. Add a hook-friendly CLI or API surface that emits only bounded coordination deltas.
3. Keep the first slice aligned with the existing `view` categories instead of introducing a second classification model.
4. Preserve compactness so downstream surfaces can reuse the feed without transcript bloat.

## Verification

- coordination change detection is reusable outside the watch loop
- hook or downstream surfaces can consume bounded deltas directly
- emitted deltas stay aligned with the current dashboard and drilldown categories

## Result

Coordination changes become reusable signals instead of watch-loop-only output.
