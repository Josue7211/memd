# Phase 47.2: `v6` Token and Context Observability — Plan

## Goal

Make token burn and context pressure visible enough that operators can choose
to compact, fork, or prefer compiled knowledge before hitting limits.

## Plan

1. Keep local context observability portable across harnesses.
2. Attribute token pressure by source class instead of only by total size.
3. Surface cache cliffs and redundant rereads as warnings.
4. Keep the outputs short, actionable, and easy to scan mid-session.

## Verification

- `cargo fmt --all`
- `cargo test -p memd-client`
- current `gap` / `status` surfaces already include pressure and redundancy
  signals

## Result

Operators can see why token budget is being burned before limits hit.
