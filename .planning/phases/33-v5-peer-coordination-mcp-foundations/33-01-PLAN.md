# Plan 33-01: `v5` Peer Coordination MCP Foundations

## Goal

Expose the brokered session coordination substrate through a stable peer MCP
surface.

## Steps

1. Define MCP-facing coordination primitives around list peers, inbox, send, acknowledge, and claim state.
2. Reuse the backend-brokered message and claim routes instead of inventing a second coordination store.
3. Preserve session-qualified identity, claim ownership, and assignment semantics.
4. Keep the first slice narrow and tool-oriented before adding richer shared-task orchestration.

## Verification

- peer coordination primitives can be exercised without bundle-local JSON state
- MCP-facing contract stays aligned with backend routes and CLI behavior

## Result

Agent sessions can coordinate through `memd` natively instead of only through
CLI wrappers.
