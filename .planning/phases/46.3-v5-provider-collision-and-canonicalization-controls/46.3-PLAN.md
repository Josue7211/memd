# Phase 46.3: `v5` Provider Collision and Canonicalization Controls — Plan

## Goal

Prevent one provider or harness from silently overwriting another provider's
truth while keeping contested memory visible and lease-aware.

## Plan

1. Expand live session identity so provider, harness, machine, session, and
   workspace are explicit.
2. Preserve contested and duplicate live records rather than flattening them.
3. Keep claim and lease enforcement visible in the shared-state surfaces.
4. Surface collision warnings in summaries so operators can spot identity drift.

## Verification

- `cargo fmt --all`
- `cargo check -p memd-client -p memd-server`
- `cargo test -p memd-client -p memd-server`

## Result

The shared memory substrate can distinguish concurrent harnesses safely and
warn when the same session identity is being reused in ways that could corrupt
shared state.
