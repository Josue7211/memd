# Phase 46.3: v5 Provider Collision and Canonicalization Controls — Summary

## Goal

Keep shared `memd` safe when multiple providers or harnesses touch the same
session state.

## What Changed

1. Live session identity is canonicalized across the important axes.
   - server-side peer sessions key on session, project, namespace, workspace,
     agent, and effective_agent
   - the client can distinguish same-named sessions across different harnesses
2. Collision warnings are visible in shared-awareness output.
   - reused live session labels trigger warnings instead of silent overwrite
   - operators can see when multiple bundles or agents claim the same session
3. Ownership remains explicit.
   - claims can be transferred and released
   - the shared state keeps workspace-scoped ownership visible

## Verification

- `cargo fmt --all`
- `cargo check -p memd-client -p memd-server`
- `cargo test -p memd-client -p memd-server`

## Result

Codex, Claude, OpenClaw, and other harnesses can share memd without
silently corrupting each other's memory records.
