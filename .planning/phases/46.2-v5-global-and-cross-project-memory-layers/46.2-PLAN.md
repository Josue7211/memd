# Phase 46.2: `v5` Global and Cross-Project Memory Layers — Plan

## Goal

Make `memd` behave as one global memory system with explicit project and
cross-project overlays so the active project stays local truth while still
reading from the same canonical global substrate.

## Plan

1. Keep the global bundle root at `~/.memd` as the default add-on path.
2. Keep project bundles resolving to `<repo>/.memd` when a project root exists.
3. Preserve workspace and initiative lanes as their own layer above project
   memory but below unrelated global recall.
4. Record bootstrap source fingerprints so future import refreshes can skip
   unchanged files.
5. Update roadmap / state docs to reflect that the global/project overlay model
   is now real behavior, not just a future design.

## Verification

- `cargo fmt --all`
- `cargo check -p memd-client`
- `cargo test -p memd-client`

## Result

The global memory path is first-class, project overlays are preserved, and the
bootstrap import path now leaves behind a source registry for read-once
behavior.
