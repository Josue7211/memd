# Plan 36-01: `v5` Claim Recovery and Coordination Automation

## Goal

Add bounded coordination automation so stale sessions stop blocking shared work.

## Steps

1. Detect stale/dead ownership pressure from heartbeat and claim state.
2. Surface reclaimable claims and stalled shared tasks explicitly.
3. Add a safe recovery path for rerouting or reclaiming stalled work.
4. Keep the first slice operator-visible and session-safe before heavier autonomous routing.

## Verification

- stale sessions can be identified as blocking shared work
- reclaimable claims and stalled tasks surface clearly
- recovery paths preserve explicit ownership transitions instead of hidden reassignment

## Result

Active sessions can safely recover blocked coworking lanes instead of waiting
for manual cleanup.
