# Plan 41-01: `v5` Coordination Drilldown and Filter Views

## Goal

Make coordination inspection faster under load by adding bounded drilldown and
filter views on top of the current dashboard surface.

## Steps

1. Add a coordination detail selector to the CLI so operators can request one bounded slice instead of the full dashboard.
2. Add matching MCP drilldown support so peer agents can inspect the same focused slices.
3. Keep filters bounded around existing coordination categories rather than introducing a freeform query surface.
4. Preserve compact output so focused views remain useful during active coworking sessions.

## Verification

- operators can request one coordination slice without rereading the full dashboard
- MCP and CLI surfaces stay aligned around the same drilldown categories
- bounded filters remain compact and useful under active coworking pressure

## Result

Coordination moves from one overview page to a usable overview-plus-drilldown workflow.
