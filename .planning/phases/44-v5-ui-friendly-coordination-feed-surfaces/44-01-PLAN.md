# Plan 44-01: `v5` UI-Friendly Coordination Feed Surfaces

## Goal

Make the coordination delta model easier for richer operator surfaces to consume
without inventing a second event taxonomy.

## Steps

1. Add a cleaner response shape for bounded coordination deltas that UI consumers can read directly.
2. Keep the first slice aligned with the existing `view` categories and alert model.
3. Preserve compact pressure summaries alongside the delta payload so consumers do not need to recompute them.
4. Avoid transcript-like event feeds; keep the surface bounded and operator-oriented.

## Verification

- richer consumers can read a clean coordination delta surface directly
- the feed stays aligned with the current dashboard, drilldown, watch, and hook categories
- the response remains compact enough for operator-facing surfaces

## Result

Coordination deltas become easier to reuse in future dashboards and control planes.
