# Phase 46.5: v5 Project Init Seeding — Context

## Why This Exists

The initial project bundle should not be blank. When `memd init` can infer a
project root, it should seed from the repo's existing docs, planning files, and
any existing Claude project memory so the user gets immediately useful memory
instead of a placeholder.

## Decisions

- Prefer `<repo>/.memd` when bootstrapping inside a project.
- Seed from repo docs, planning files, and existing memory files that already
  describe the project.
- Keep a manual deeper-import path available when the first bootstrap pass is
  too thin.
- Keep the first init pass bounded so it stays fast and readable.

## Discretion Areas

- How much bootstrap text should be included in the placeholder memory file.
- Which source files count as the default seeding surface versus manual import
  surface.
- Whether bootstrap output should highlight what was missing as much as what
  was imported.

## Deferred Ideas

- Full repo graph import.
- Automatic deep import without user confirmation.
- Any more advanced bootstrap UI.
