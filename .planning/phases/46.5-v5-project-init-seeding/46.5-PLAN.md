# Phase 46.5: `v5` Project Init Seeding — Plan

## Goal

Make `memd init` create a useful project bundle on first setup by seeding from
the repo's existing memory-bearing files instead of leaving the project blank.

## Plan

1. Resolve project-root inits to the local `.memd` bundle.
2. Seed from the existing project docs and memory files that already capture
   project context.
3. Record the imported sources once so later refreshes can stay delta-driven.
4. Leave a separate manual path for deeper imports when the default seed is
   not enough.

## Verification

- `cargo fmt --all`
- `cargo test -p memd-client`
- `memd init --project-root <repo>` produces a seeded local project bundle.

## Result

First-time project setup yields an immediately useful memd bundle instead of a
blank placeholder.
