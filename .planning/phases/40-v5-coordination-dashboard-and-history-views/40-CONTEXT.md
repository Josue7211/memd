# Phase 40 Context: `v5` Coordination Dashboard and History Views

## Why This Phase Exists

`memd` now has coordination receipts, but the next gap is presentation: active
operators still need a cleaner history-oriented view for recent coworking
transitions and open coordination pressure.

## Inputs

- coordination inbox
- receipts and audit trail
- policy conflicts and boundary recommendations
- stale-session recovery and shared task orchestration

## Constraints

- keep views compact and operator-friendly
- avoid transcript-style dumps
- stay compatible with current CLI and MCP usage

## Target Outcome

The next phase should add cleaner dashboard/history views over the existing
coordination substrate so operators can inspect current and recent coworking
state more quickly.
