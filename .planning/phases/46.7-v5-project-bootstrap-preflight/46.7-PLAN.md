# Phase 46.7: `v5` Project Bootstrap Preflight — Plan

## Goal

Make project bootstrap report what scaffolding exists before init and keep
`memd` useful even when the repo has no full agent scaffold.

## Plan

1. Detect common project scaffolding before init.
2. Keep missing scaffolding as a warning, not a blocker.
3. Seed from whatever project files are available.
4. Offer a deeper import path only when the default seed is too thin.

## Verification

- `cargo fmt --all`
- `cargo test -p memd-client`
- bootstrap output clearly reports found/missing scaffolding when a repo is
  seeded

## Result

Users can run the front door once and still get a useful bundle even in thin
repos.
