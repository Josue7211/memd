# Phase 46.6: v5 Unified Memd Front Door — Context

## Why This Exists

The user should remember one Codex command: `$memd`. The front door then
routes to init or reload based on repo state and user intent so the bootstrap
split stays internal instead of becoming part of the mental model.

## Decisions

- Keep the unified `memd` skill as the default remembered surface.
- Keep `memd-init` and `memd-reload` as explicit subflows for control, not the
  primary user-facing model.
- Make the skill explain what happened in one short sentence.
- Avoid adding a separate status or doctor step for the happy path.

## Discretion Areas

- How much routing detail should be visible in the skill output.
- Whether the unified front door should mention global/project overlay details
  when the user does not ask for them.
- Whether the command should prefer init or reload when repo state is ambiguous.

## Deferred Ideas

- A richer memory browser.
- Any provider-specific front door beyond Codex.
- More advanced bootstrap diagnosis for users who ask for it explicitly.
