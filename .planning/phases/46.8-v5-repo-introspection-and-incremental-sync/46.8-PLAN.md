# Phase 46.8: `v5` Repo Introspection and Incremental Sync — Plan

## Goal

Make repo files into structured memory input and keep them current with a
delta refresh path that only rereads sources when they actually change.

## Plan

1. Seed project bundles from the repo files that already describe the project.
2. Store fingerprints for those imported sources in a registry.
3. Refresh the imported project memory by comparing metadata before rereading
   unchanged files.
4. Keep the layered memory model intact while allowing project and cross-
   project state to stay queryable together.

## Verification

- `cargo fmt --all`
- `cargo check -p memd-client`
- `cargo test -p memd-client`

## Result

Memd can keep project memory current without forcing a fresh bootstrap sweep
every time the repo changes.
