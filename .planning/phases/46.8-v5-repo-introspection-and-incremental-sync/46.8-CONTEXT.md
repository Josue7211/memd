# Phase 46.8: v5 Repo Introspection and Incremental Sync — Context

## Why This Exists

Repo files should be treated as structured memory input, not just bootstrap
material. Memd also needs a delta refresh path so changed files get reimported
without forcing a full repo reread on every refresh.

## Decisions

- Import repo docs, planning files, and config with source metadata.
- Record a fingerprinted source registry once so unchanged files can be
  skipped later.
- Use file metadata and content hashes to detect changes before rereading
  sources.
- Keep global, project, and cross-project memory queryable together without
  flattening them.

## Discretion Areas

- Which files belong in the default refresh set versus manual import.
- How much diff detail should appear in the generated refresh section.
- Whether git history should be surfaced in the registry or only in gap
  reporting.

## Deferred Ideas

- Full graph extraction from the entire repo.
- Any learned refresh prioritization.
- More advanced repo-wide confidence scoring.
