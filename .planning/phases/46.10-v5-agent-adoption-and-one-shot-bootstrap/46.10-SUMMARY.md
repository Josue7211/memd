# Phase 46.10: v5 Agent Adoption and One-Shot Bootstrap — Summary

## Goal

Make the major runtimes start from the same shared memory substrate.

## What Changed

1. Codex, Claude, and OpenClaw all have memd bootstrap surfaces.
   - hooks and skills load the shared memory
   - workspace bootstrap docs point agents at the canonical memd bundle
2. The shared memory is only imported once.
   - project bootstrap seeds from existing files
   - source registry fingerprints the imported sources
3. The user-facing path stays simple.
   - `$memd` routes init vs reload
   - the user doesn’t have to learn separate setup rituals for each harness

## Verification

- hook checks and installed skill surfaces already point at memd

## Result

The common agent runtimes all start from the same shared memory substrate
instead of divergent local-only state.
