# Phase 46.10: v5 Agent Adoption and One-Shot Bootstrap — Context

## Why This Exists

The common agent runtimes should all start from the same shared memd substrate
instead of each forcing the user to rebuild context manually. The first useful
session should happen once, then future sessions should load the existing
memory automatically.

## Decisions

- Wire Codex, Claude, and OpenClaw startup paths to memd.
- Keep bootstrap import bounded so it reads existing memory once.
- Preserve a simple first-use path: one obvious command, not multiple setup
  steps.
- Keep the shared memd substrate as the source of truth across harnesses.

## Discretion Areas

- How much the adoption surfaces should explain on startup versus staying
  silent after success.
- Whether each harness should surface a distinct success message or a shared
  one.
- How to handle runtimes that start before their hooks are active.

## Deferred Ideas

- Any deeper runtime-specific bootstrap policy beyond the current integration
  hooks.
- Additional harnesses beyond the current major three.
- More advanced onboarding or education screens.
