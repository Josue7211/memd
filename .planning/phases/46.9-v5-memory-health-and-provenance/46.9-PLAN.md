# Phase 46.9: `v5` Memory Health and Provenance — Plan

## Goal

Give operators a clear view of which memory facts can be trusted, which ones
are stale or contested, and where each fact came from.

## Plan

1. Keep source quality, confidence, and provenance on memory records.
2. Surface stale and conflicting memory in status and maintenance views.
3. Make drift visible instead of silently replacing contested facts.
4. Keep provenance available for both operators and downstream automation.

## Verification

- `cargo fmt --all`
- `cargo test -p memd-client -p memd-server`
- existing maintenance / status outputs already expose stale and low-confidence
  state

## Result

Users can inspect where a memory fact came from and whether it should be
trusted.
