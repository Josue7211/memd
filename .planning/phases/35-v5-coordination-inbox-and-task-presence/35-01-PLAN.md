# Plan 35-01: `v5` Coordination Inbox and Task Presence

## Goal

Build a compact coordination view that combines inbox, task state, and peer
presence for active coworking sessions.

## Steps

1. Add a coordination inbox surface that joins direct peer messages with shared task ownership and request state.
2. Expose waiting help/review/assignment work without forcing agents to poll several separate primitives.
3. Keep the first slice compact and session-scoped so it can feed resume, handoff, and MCP workflows.
4. Preserve explicit ownership and inspectability instead of hiding coordination behind opaque automation.

## Verification

- a session can inspect incoming coordination pressure from one command or route
- help/review/assignment tasks surface clearly with owner and requester identity
- the new view remains aligned across CLI and MCP-facing coordination surfaces

## Result

Agent sessions get a compact coordination inbox instead of manually stitching
together messages, tasks, and liveness.
