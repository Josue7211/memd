# Plan 34-01: `v5` Shared Task Orchestration

## Goal

Build a first shared-task orchestration layer on top of the brokered peer
coordination substrate.

## Steps

1. Add a task-oriented coordination model that groups claims, assignment, help, and review around named shared tasks.
2. Reuse the existing backend and MCP bridge instead of inventing separate orchestration state outside `memd`.
3. Surface task ownership, status, and transfer paths clearly enough for simultaneous coworking across terminals.
4. Keep the first slice deterministic and inspectable before adding heavier automation.

## Verification

- shared tasks can be listed and inspected without reconstructing state from raw messages
- assignment/help/review flows can target a shared task instead of only loose scopes
- ownership remains session-qualified and claim-safe across CLI and MCP surfaces

## Result

Agent sessions can coordinate around explicit shared tasks instead of only
primitive messages and scope claims.
