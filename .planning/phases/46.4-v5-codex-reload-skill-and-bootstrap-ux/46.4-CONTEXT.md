# Phase 46.4: v5 Codex Reload Skill and Bootstrap UX — Context

## Why This Exists

Open Codex sessions still need an obvious way to load memd immediately without
waiting for the next hook-triggering tool use. The user-facing bootstrap should
be one remembered command that forces the current session to refresh from the
global bundle, then the project overlay if one exists.

## Decisions

- Keep `memd-reload` as the explicit Codex-native reload surface.
- Keep the shell command and the Codex skill aligned so the same bootstrap
  works in both terminals and sessions.
- Default reload to the global bundle first, then layer the current repo bundle
  only when one exists.
- Keep the bootstrap surface compact and session-oriented, not a full browser or
  memory explorer.

## Discretion Areas

- How much status text should be shown when no local project bundle exists.
- Whether reload should auto-create a project bundle or leave that to init.
- How aggressively the skill should remind the user about the project overlay.

## Deferred Ideas

- Any richer memory browsing UI.
- Automatic session-refresh timers.
- Provider-specific reload UX beyond the current Codex surface.
