# Phase 46.4: `v5` Codex Reload Skill and Bootstrap UX — Plan

## Goal

Give already-open Codex sessions one obvious command to load memd immediately
so the current session can adopt the canonical global memory and any local
project overlay without waiting for the next tool call.

## Plan

1. Keep the Codex skill surface focused on reload, not full browsing.
2. Keep shell and skill reload paths aligned so the same behavior works in both
   places.
3. Refresh the global bundle first, then the project overlay if present.
4. Keep the status text short enough that users know what happened at a glance.

## Verification

- `cargo fmt --all`
- `cargo test -p memd-client`
- Codex skill / shell invocations load the bundle without a manual multi-step
  setup dance.

## Result

Open sessions can be reloaded into memd with one remembered command.
