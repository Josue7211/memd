# Plan 20-01: `v4` Short-Term Checkpoints

## Goal

Make current-task memory capture fast enough to use routinely.

## Steps

1. Add a dedicated `checkpoint` CLI command.
2. Translate checkpoint requests into the existing `remember` flow.
3. Default checkpoints to:
   - `status` kind
   - `project` scope
   - short-term tags
   - one-day TTL
4. Document the command in the README.
5. Add regression coverage for the default translation behavior.

## Verification

- `cargo test -q -p memd-client`
- `cargo test -q`

## Result

Short-term state becomes easy to capture without creating a parallel storage
system.
