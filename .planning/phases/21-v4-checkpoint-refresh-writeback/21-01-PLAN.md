# Plan 21-01: `v4` Checkpoint Refresh Writeback

## Goal

Make checkpoint writes immediately visible to agent-facing memory files.

## Steps

1. Refresh bundle resume after a successful checkpoint write.
2. Keep the refresh on the fast local path with semantic fallback disabled.
3. Regenerate bundle memory files from that refreshed snapshot.
4. Update README guidance so operators know checkpoints refresh memory files.
5. Record the phase completion in `.planning`.

## Verification

- `cargo test -q -p memd-client`
- `cargo test -q`

## Result

Short-term memory writeback becomes immediate instead of multi-step.
