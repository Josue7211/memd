# Plan 38-01: `v5` Branch and Scope Recommendations

## Goal

Recommend cleaner work boundaries for simultaneous peer sessions before work
overlaps.

## Steps

1. Derive branch and scope recommendations from task coordination mode, existing claims, and active ownership.
2. Surface recommendations through compact CLI and MCP coordination views.
3. Keep the first slice advisory instead of automatically mutating git state.
4. Preserve explicit operator control over final branch and scope decisions.

## Verification

- active tasks can surface cleaner branch/scope suggestions
- recommendations stay compatible with existing claim and task ownership flows
- the new layer remains advisory and inspectable instead of silently mutating git state

## Result

Simultaneous sessions can split work more cleanly before stepping on each
other’s implementation boundaries.
