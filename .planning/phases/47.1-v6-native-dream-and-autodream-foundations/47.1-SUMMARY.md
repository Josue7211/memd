# Phase 47.1: v6 Native Dream and Autodream Foundations — Summary

## Goal

Recognize dream/autodream as native lifecycle concepts built on top of the
existing consolidation substrate.

## What Changed

1. Native consolidation already exists.
   - `memd consolidate` is the core lifecycle primitive
   - it keeps memory promotion and accepted-signal flow inside the native CLI
2. Wrapper surfaces stay thin.
   - skills and shell hooks can schedule or invoke consolidation
   - they do not need to own the core lifecycle logic
3. The first slice stays clean.
   - boundaries, flow, and compatibility are the main concerns
   - no extra UI or duplicate consolidation engine was introduced

## Verification

- native `consolidate` command exists in the CLI

## Result

Dream and autodream now have a native `memd` lifecycle substrate to build on.
