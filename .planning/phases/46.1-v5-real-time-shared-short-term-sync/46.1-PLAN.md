# Phase 46.1: `v5` Real-Time Shared Short-Term Sync — Plan

## Goal

Turn `memd-server` into the canonical live state hub for shared short-term
memory so agents on different machines and harnesses can cowork from one
shared operational view instead of drifting into local-only truth.

## Plan

1. Extend the shared memory schema with explicit peer-session records that
   carry live identity, workspace, presence, and heartbeat metadata.
2. Persist peer sessions in the server and expose list/upsert routes so
   runtime state can be published and observed centrally.
3. Wire client heartbeats and shared-awareness rendering to prefer the server
   view for workspace-scoped short-term state.
4. Keep the sync payload compact and bounded: focus, blockers, next step,
   claims, ports/base URLs, handoff/review/help requests, and presence.
5. Make runtime bootstrap surfaces for Codex, Claude, and OpenClaw read the
   shared state so the same short-term lane is visible across harnesses.

## Verification

- `cargo fmt --all`
- `cargo check -p memd-client -p memd-server -p memd-schema`
- `cargo test -p memd-client`
- `cargo test -p memd-server`
- runtime hook syntax checks for Codex and Claude bootstrap scripts

## Result

The phase is complete when the shared short-term lane is server-backed,
workspace-aware, and visible to the main agent runtimes without requiring
manual transcript replay.
