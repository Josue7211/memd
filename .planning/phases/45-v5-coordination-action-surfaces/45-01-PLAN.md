# Plan 45-01: `v5` Coordination Action Surfaces

## Goal

Let richer operator surfaces trigger bounded coordination actions through the
same shared model they already use for inspection.

## Steps

1. Expose the first bounded coordination actions through cleaner operator-facing surfaces.
2. Keep the first slice focused on acknowledgement, assignment, and recovery actions instead of a broad control plane.
3. Reuse the same inbox, request, recovery, policy, and history categories instead of inventing a UI-only contract.
4. Preserve explicit operator control and inspectability around every action.

## Verification

- richer operator surfaces can trigger bounded coordination actions directly
- action surfaces stay aligned with the current inspection and feed categories
- explicit operator control remains visible around each action

## Result

Coordination moves from inspection-only richer surfaces toward bounded actionability.
