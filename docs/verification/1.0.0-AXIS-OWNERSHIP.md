## Cross-phase scope resolution (post-0.1.0 overlaps)

### Overlap 1: Telemetry substrate (V14 ↔ V15 ↔ V20)

| Milestone | Scope | Axis action |
|---|---|---|
| V14 | Anonymized per-user telemetry pipeline + cost ledger integration | OWNS TE lift 7→8 |
| V15 | Self-tuning compiler consuming V14 telemetry | OWNS TE lift 8→9 |
| V20 | Info-theoretic optimality proof (no token removable) | OWNS TE lift 9→10 |

All three touch TE. Each owns one delta. No phantom double-claim possible:
V15 regenerator must verify V14 telemetry present; V20 regenerator must
verify V15 self-tuning operational; each claims its own delta only.

### Overlap 2: CR graph (V18 ↔ V19)

| Milestone | Scope | Axis action |
|---|---|---|
| V18 | Multi-hop correction chains + silent correction detection (≥0.90 precision / ≥0.85 recall) | OWNS CR lift 8→9 |
| V19 | ZK proofs + third-party replayable corrections | OWNS CR lift 9→10 |

V19 depends on V18 graph primitive but claims distinct delta (replay-
verifiability is the 9→10 step, not detection quality).

### Overlap 3: Cross-harness (V16 ↔ V17)

| Milestone | Scope | Axis action |
|---|---|---|
| V16 | Cross-device sync (same user, multiple devices) | OWNS CH lift 8→9 |
| V17 | Cross-user federation (multiple users, marketplace) | OWNS CH lift 9→10 |

Device-scale CH (V16) and user-scale CH (V17) are distinct dimensions
of the same axis. Each owns one delta.

  `axes_lifted` / `axes_integrated_with` fields.

  score higher than this table permits for that milestone.

  fix by demoting one to `INT` per overlap-resolution tables above.
