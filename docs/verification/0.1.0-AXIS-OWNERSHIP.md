# 0.1.0 Axis Ownership

> Binding per-axis-delta ownership table. Each axis delta has exactly one
> owning milestone. Every other milestone that touches the axis is an
> `integrator` — it must not regenerate the score or claim additional lift
> on that axis without first updating this table.

## Why single ownership matters

Two milestones claiming the same axis lift = phantom composite math.
V4's old spec failed this on procedural_reuse; fixing it one milestone at
a time leaves V5-V10 free to repeat the mistake. This table prevents
re-occurrence.

## Cross-phase scope resolution (known overlaps)

Three overlaps identified during V4 audit. Resolutions are binding; any
milestone spec that contradicts needs review revision.

### Overlap 1: Correction E2E (C4 ↔ B5 ↔ A7)

| Phase | Scope | Axis action |
|---|---|---|
| C4 (V4)  | Correction ingestion + supersede mechanics | OWNS CR lift 1→4 |
| B5 (V5)  | CorrectionPropagation bench suite measuring C4 | NO axis credit — measures what V4 already claimed |
| A7 (V7)  | Behavior-change E2E (future session uses corrected value, not just storage) | OWNS CR lift 4→5 |

### Overlap 2: Working-Context Compiler (D4 ↔ D6)

| Phase | Scope | Axis action |
|---|---|---|
| D4 (V4)  | Compiler + kinds-coverage + cost ledger + 4-layer cap | OWNS TE lift 2→4 |
| D6 (V6)  | Same compiler applied to public-bench inputs | NO axis credit — integrates; wins go to RR + TP for V6 |

### Overlap 3: Progressive-Depth Recall (E4 ↔ D5 ↔ E6)

| Phase | Scope | Axis action |
|---|---|---|
| E4 (V4)  | Depth contract + FTS5+RRF + query sanitization | INTEGRATES with V4 SC + TE lifts |
| D5 (V5)  | ProgressiveDepth bench in substrate suite | NO axis credit — measurement |
| E6 (V6)  | Progressive-depth routing on bench inputs | NO axis credit — integrates; wins to RR |

## Enforcement

- Every milestone doc must cite this table in its `axes_lifted` / `axes_integrated_with` fields.
- Every G-gate scorecard regenerator must refuse to write an axis score
  higher than this table permits for that milestone.
- Contract violations (two milestones claiming same delta) fail review;
  fix by demoting one to `INT` per overlap-resolution table.
