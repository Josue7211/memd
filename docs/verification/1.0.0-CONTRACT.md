# memd 1.0.0 Release Contract — 10-STAR Ceiling

> Post-0.1.0 push. 1.0.0 ships when the 10-STAR scorecard reads
> **composite 10.00, every axis 10/10**. This is the full ceiling that
> `docs/verification/MEMD-10-STAR.md` was designed to measure. 0.1.0
> proves SOTA; 1.0.0 proves dominance that the scorecard was calibrated
> to recognize.

## Why 1.0.0 = 10/10

The 10-STAR scorecard in `MEMD-10-STAR.md` was not aspirational filler —
it was designed to be hit. 0.1.0 lands every axis at ≥7 (SOTA floor);
1.0.0 lands every axis at 10 (ceiling). Between 0.1.0 and 1.0.0, memd
accumulates the real-user telemetry, multi-month usage data, and
cross-user interaction patterns that 10/10 requires. 10/10 cannot be
fabricated pre-release — it has to be earned from deployment.

Theory binding: `docs/theory/MEMD-SOTA-THEORY.md` §"7 axes — what 10/10
means". Every 10/10 target in that doc is a claim this contract commits
to proving before the 1.0.0 tag.

## Release gate (1.0.0)

1. **Composite 10.00** on the 10-STAR scorecard (zero-generosity regrade).
2. **Every axis 10/10.** No ≥7 floor — ceiling enforced.
3. **Zero blocker-severity backlog** across all axes and all phases.
4. **Reproducible proof run** in `docs/verification/release-1-0-0/` with:
   - per-axis NDJSON harness evidence
   - third-party replay traces (signed + ZK proofs where applicable)
   - dated external reviews (two independent reviewers per axis minimum)
   - multi-month real-user telemetry snapshots (anonymized)
5. **Head-to-head dominance proof** — ≥10pp margin on every public
   bench (LoCoMo, LongMemEval, MemBench, ConvoMem) + at least one
   memd-authored harder bench that published SOTA competitors fail on.
6. **Info-theoretic TE proof** — no token in the compiled context can
   be removed without measurable downstream task quality degradation.
7. **Cross-user federation proof** — thousands-of-users scale test
   with per-user isolation preserved, routine marketplace live with
   trust+provenance, zero data leakage incidents in proof window.

## Per-milestone scope (binding summary)

- **V14 Telemetry + Observability Foundation** — TE 7→8. Real-user
  telemetry substrate: cost ledger backed by anonymized per-user
  per-harness usage data. Foundational; unlocks V15 self-tuning and
  V20 info-theoretic TE proofs. No other axis lifts.
- **V15 Self-Tuning Compiler** — TE 8→9. Compiler learns per-user
  per-harness what budget maximizes downstream task quality, adjusts
  automatically. Builds on V11 dynamic compiler + V14 telemetry.
- **V16 Cross-Device Sync at Scale** — SC 9→10, CH 8→9. CRDT-based
  multi-device sync. Dormant-project recovery at months-long horizon
  with zero quality delta. Cross-device replay produces identical
  behavior turn-for-turn.
- **V17 Cross-User Routine Economy** — PR 9→10, CH 9→10. Routine
  marketplace with discovery, trust, provenance, per-user reputation.
  Parameterized routine generalization (infer variable bindings from
  example traces). Federation at scale: thousands of users, per-user
  isolation + explicit sharing.
- **V18 Correction Graph** — CR 8→9. Correction graph across all memory
  with full multi-hop chains. Silent correction detection reaches
  ≥0.90 precision / ≥0.85 recall. Third-party replay from export
  reproduces correction applications.
- **V19 Zero-Knowledge Provenance + Full Cryptographic Audit** — TP
  9→10, CR 9→10. Tamper-evident audit trail. ZK proofs: user can
  prove a correction was applied without revealing correction content.
  Compliance-grade audit UI. Multi-party attestation.
- **V20 Info-Theoretic TE + Bench Ceiling** — RR 9→10, TE 9→10.
  Info-theoretic optimal compiler (no token removable without quality
  loss). Dominates every public bench by ≥10pp. Publishes new harder
  benches that existing SOTA systems fail on. Generalizes to zero-shot
  domains.

## Non-goals for 1.0.0 (deferred to 2.0.0+)

- AGI-like planning memory (memd stays substrate-level; planning lives
  in the harness)
- Memory-backed tool invocation chains (harness territory)
- Human-in-the-loop correction orchestration UI beyond what V18 ships
- Regulated-industry compliance audits (HIPAA/SOC2/PCI) — deployment
  partner work, not substrate work
- Hardware-assisted memory isolation (TEE, enclaves) — post-1.0.0

## Relationship to 0.1.0

0.1.0-CONTRACT.md stands. V1-V13 remain the 0.1.0 release gate. V14-V20
are post-0.1.0 milestones — they do **not** block 0.1.0 shipping.
0.1.0 is the SOTA launch; 1.0.0 is the SOTA ceiling.

If 0.1.0 ships and real-user telemetry reveals that V14-V20 scoping is
wrong (e.g., users don't use cross-user routines enough to justify the
effort), this contract is revised. The chain composite-math above is
correct against V13 baseline, but axis definitions at 10/10 may shift
if theory shifts. Contract is living until 1.0.0 tag lands.
