# memd 0.1.0 Release Contract

> Authoritative gate for the V1–V13 pipeline end-state. If this contract is not
> satisfied, 0.1.0 does not ship. Every milestone (V4–V13) contributes a
> documented axis delta; every delta is reversible until G-gate harness proof
> lands. **0.1.0 ships at SOTA (state-of-the-art), not at production floor** —
> the THEORY is "best SOTA memory OS for any harness" (see MEMD-SOTA-THEORY.md).

## Why a contract

Composite scores can be gamed by one heavy axis (raw_retrieval at 6/10 can
hide procedural_reuse at 1/10). The 0.1.0 gate enforces **breadth** through
per-axis floors in addition to the composite, so every pillar clears the
"user gets value" bar, not just the ones that are easy to lift.

## Gate

**0.1.0 ships only when all five conditions hold simultaneously:**

1. **Composite ≥ 8.0** on the 10-STAR scorecard, zero-generosity regrade rules.
   (V13 target: 8.50. Floor 8.0 allows one axis miss without blocking.)
2. **Every axis ≥ 7/10.** SOTA floor — no axis may sit below 7 at release.
3. **Zero blocker-severity backlog** tagged with any 10-STAR axis label.
4. **Reproducible proof run** in `docs/verification/release-0-1-0/` with
   per-axis evidence trail (harness NDJSON + dated human review).
5. **Head-to-head SOTA proof** — on at least one public benchmark per axis
   (where applicable), memd beats published SOTA by ≥5pp margin with dated
   transcript, or is explicitly parity-with-margin for axes where no public
   bench exists (e.g. procedural_reuse, trust_provenance).

The release commit runs the G13 harness against the current main and
regenerates MEMD-10-STAR.md in place. If any axis regresses on regeneration,
0.1.0 does not tag. **V10 is not the release gate** — V10 closes the
production-floor milestone (composite 6.40, every axis ≥3). V11-V13 push to
SOTA (composite 8.50, every axis ≥7). Release tag lands at V13 close only.

## Rules

- **No axis credit without G-harness proof.** Code landing ≠ axis lift.
  A milestone spec that claims +2 on an axis without a G-harness assertion
  is invalid and blocks milestone closure.
- **Demotion is allowed.** If a harness run regresses an axis, roll the
  number back and file a recovery phase. Do not rewrite the contract to
  hide the regression.
- **No compensating lifts.** One axis +1 cannot offset another's −1 within
  a milestone. Both must be addressed.
- **Floor violations block release even if composite passes.** A composite
  of 6.1 with procedural_reuse at 2 does not ship 0.1.0.
