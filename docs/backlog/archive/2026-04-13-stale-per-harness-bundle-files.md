# Stale Per-Harness Bundle Files

- status: `closed`
- found: `2026-04-13`
- scope: memd-client (init_runtime), .memd bundle

## Summary

16 per-harness markdown files from the pre-10-star model exist in `.memd/agents/`.
Not generated by current init code, not referenced in any code path. The 10-star
model uses 3 universal files (wake.md, mem.md, events.md) per harness instead.

## Symptom

- `.memd/agents/` contains 16 dead files (~93KB)
- Confusing for anyone reading the bundle structure
- `HARNESS_BRIDGES.md` is still written by `bootstrap_runtime.rs:407` — may also be stale

## Root Cause

- Migration from per-harness files to universal 3-file model was incomplete
- Old files were never cleaned up
- `.memd/` is gitignored so these are local-only, won't regenerate

## Fix Shape

- Delete 16 files from `.memd/agents/`:
  `AGENT_ZERO_MEMORY.md`, `AGENT_ZERO_WAKEUP.md`,
  `CLAUDE_CODE_EVENTS.md`, `CLAUDE_CODE_MEMORY.md`, `CLAUDE_CODE_WAKEUP.md`,
  `CODEX_EVENTS.md`, `CODEX_MEMORY.md`, `CODEX_WAKEUP.md`,
  `HERMES_MEMORY.md`, `HERMES_WAKEUP.md`,
  `OPENCLAW_EVENTS.md`, `OPENCLAW_MEMORY.md`, `OPENCLAW_WAKEUP.md`,
  `OPENCODE_EVENTS.md`, `OPENCODE_MEMORY.md`, `OPENCODE_WAKEUP.md`
- Audit `HARNESS_BRIDGES.md` — check if `bootstrap_runtime.rs:407` still needs it
- Add cleanup step to `memd init` that removes these if found

## Evidence

- `git ls-files .memd/` returns empty — gitignored
- Grep for filenames in crates returns zero hits (except HARNESS_BRIDGES)
