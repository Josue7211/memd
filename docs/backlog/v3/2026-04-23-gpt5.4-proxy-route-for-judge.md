# gpt-5.4 proxy route for public-bench judge + generator

## Resolution summary (2026-04-23)

**Not an infra gap — a wrong-URL diagnosis error on my part.** codex-lb has
been running `gpt-5.4` + `gpt-5.4-mini` (plus `gpt-5.3-codex`, etc.) at
`http://127.0.0.1:2455` the entire time, authenticated by `$CODEX_LB_API_KEY`
(`sk-clb-...`). I was probing the wrong proxy — the openclaw LiteLLM at
`$OPENAI_BASE_URL` (port 4000), authenticated by `$OPENAI_API_KEY`
(`sk-mc-...`), is a **different** service with a narrower model allowlist
(`gpt-5-codex`, `haiku-manager`, `kimi-research`, `claude-fallback`).

## How to run the memd canonical sweep

memd's bench harness reads `$OPENAI_BASE_URL` + `$OPENAI_API_KEY` verbatim
(see `crates/memd-client/src/benchmark/full_eval.rs:142-148`). Override
inline to route through codex-lb:

```
OPENAI_BASE_URL=http://127.0.0.1:2455/v1 \
OPENAI_API_KEY=$CODEX_LB_API_KEY \
MEMD_BENCH_JUDGE_BUDGET_USD=0 \
/tmp/memd-target/release/memd benchmark public longmemeval \
  --full-eval --limit 500 --dataset-root .memd/benchmarks/datasets/longmemeval/longmemeval_s_cleaned.json \
  --retrieval-backend memd
```

`--generator-model` + `--grader-model` default to `gpt-5.4` post-2026-04-23;
no flags needed. Cost is $0 (codex-lb is flat-rate OAuth fanout — see the
pricing table carve-out at `crates/memd-client/src/benchmark/public_benchmark.rs:1304`).
