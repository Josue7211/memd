## C4.1 scope warning ā€” read this before touching schema

C4.1 per `docs/phases/v4/phase-c4-plan.md` adds a `Correction` variant
to `MemoryKind` plus four provenance fields. The non-obvious cost:

**106 `MemoryItem` struct-literal sites across 30 files.** Rust
requires all fields specified in struct literals; `#[serde(default)]`
only affects deserialization. Every literal site must be touched.

### Pre-committed nesting decision

**Land the provenance fields as
`correction_meta: Option<CorrectionMetadata>` rather than four flat
fields on `MemoryItem`.**

Why: flat fields multiply the literal-edit volume (~318 lines vs
~106). Nested option means literal sites only add one line
(`correction_meta: None,`) and only `Correction`-emitting paths need
to construct the metadata struct.

How to apply:
- Add `struct CorrectionMetadata { corrects_id, source_turn,
  captured_by, confidence }` next to `MemoryItem`.
- Add `pub correction_meta: Option<CorrectionMetadata>` to
  `MemoryItem`.
- `grep -rn "MemoryItem {" crates/ tests/ | wc -l` should return
  ~106. Use a mechanical find+replace pass ā€” this is not a
  judgement-heavy edit.
- Add `#[serde(default, skip_serializing_if = "Option::is_none")]` so
  old serialized forms round-trip cleanly.

This is too large to commit mid-session without risking a broken
tree. Budget it as its own atomic commit.
