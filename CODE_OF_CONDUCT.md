# Code of Conduct

## Our Standard

Contributors to `memd` are expected to keep discussion technical, respectful,
and usable for other people working in public.

Examples of expected behavior:

- discuss code and product decisions directly
- challenge ideas with evidence and specifics
- keep review feedback actionable
- assume other contributors are trying to improve the project

Examples of unacceptable behavior:

- harassment, insults, or personal attacks
- deliberately hostile or exclusionary language
- publishing private information without permission
- derailing technical work with repeated bad-faith behavior

## Scope

This applies to repository discussions, issues, pull requests, and other
project spaces managed for `memd`.

## Enforcement

Project maintainers may remove comments, close discussions, or restrict access
for behavior that makes the project harder to work on safely in public.

## Reporting

If conduct becomes a project problem, contact the maintainers through the
private channel used for project administration.
